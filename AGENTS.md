# AGENTS.md — Go Blueprint Reference

**Language Rule**: All code, comments, commit messages, PR descriptions, and documentation must be written in English. No exceptions.

---

## 1. Project Identity

`github.com/eduardo-sl/go-blueprint` is a reference implementation of idiomatic Go
service architecture — DDD aggregates, explicit CQRS (no mediator), Clean Architecture
dependency rules, and a transactional outbox — built on Go 1.26. The single domain in
scope is Customer management; everything else (Auth, EventLog, Cache, Worker, Outbox)
exists to show the patterns, not to model a real business. This is a reference
architecture, not a framework: no generated DI, no reflection magic, no bus — every
dependency is wired by hand in `cmd/api/main.go`. The HTTP transport is Echo v4; the
database layer is `sqlc`-generated code over `pgx/v5`; the SQLite event log uses
`modernc.org/sqlite` (pure Go, no CGo).

## 4. Architecture Rules (non-negotiable)

1. Interfaces are defined at the consumer, not the implementor — `customer.Repository` lives in `internal/customer/`, not in `internal/platform/database/postgres/`.
2. Dependency arrows point inward: `platform/database/postgres/` imports `customer` and `auth` domain types; domain packages never import infra packages.
3. There is no mediator, command bus, or event bus — service methods are called directly; async is achieved with goroutines and worker pool, not abstraction.
4. Every error is wrapped with caller context using `fmt.Errorf("pkg.Type.Method: %w", err)`.
5. `panic()` is forbidden in domain, service, and handler code — only permissible in `main()` for fatal startup failures; in practice `main()` uses `os.Exit(1)` after logging.
6. Business validation lives in domain constructors (`customer.New()`, `customer.Update()`) and domain service methods — not in handlers; handlers validate request shape with `go-playground/validator`, then delegate.
7. Errors are logged once, at the handler boundary; lower layers wrap and return.
8. The `customer` package and the `auth` package are isolated bounded contexts — they must not import each other.
9. When a second database context is added (MongoDB for Product), it must be an independent package with its own repository interface — sharing the Postgres pool or domain types across contexts is forbidden.
10. Extensions (cache, worker, outbox) are opt-in via environment variables — empty `REDIS_ADDR` disables cache with automatic `NoopCache` fallback.
11. `sqlc`-generated files (`*.sql.go`, `models.go`, `db.go`) are never edited by hand — re-run `make generate` after changing `queries/*.sql`.
12. Mocks are generated by `mockery` from interfaces — hand-written mocks are not permitted for new work.

## 7. Key Patterns — Canonical Implementations

### Cache-Aside

- **Interface + sentinel**: `internal/platform/cache/cache.go` → `Cache` interface, `ErrCacheMiss`
- **Redis impl**: `internal/platform/cache/redis.go` → `RedisCache`; pings on `NewRedisCache`; returns error on failure
- **Noop fallback**: `cache.NoopCache{}` — used when `cfg.RedisAddr == ""`; every `Get` returns `ErrCacheMiss`; `Set`/`Delete` are no-ops
- **Consumer**: `internal/customer/query.go` → `CachedQueryService.GetByID()` and `.List()`
- **Cache keys**: `"customer:<uuid>"` for single records; `"customer:list"` for the list
- **TTLs**: single record = `cfg.CacheTTL` (default 5m); list = `max(ttl/5, 1m)` (default 1m)
- **Invalidation**: `internal/customer/service.go` → `Service.invalidate()` deletes both keys on `Update` and `Remove`; failures are logged at WARN and never propagated
- **Degradation contract**: Redis error on `Get` (not `ErrCacheMiss`) → log WARN → fall through to DB → never return 5xx; corrupted (non-unmarshalable) entries are evicted and fall through to DB

### Transactional Outbox

- **Message type**: `internal/outbox/outbox.go` → `OutboxMessage`
- **Store interface**: `internal/outbox/store.go` → `OutboxStore`; `SaveTx` takes `pgx.Tx` — calling outside a transaction defeats the pattern
- **Postgres impl**: `internal/outbox/postgres.go` → `PostgresOutboxStore`; `FetchUnprocessed` uses `FOR UPDATE SKIP LOCKED` — do not remove; prevents duplicate delivery across replicas
- **Poller**: `internal/outbox/poller.go` → `Poller.Run(ctx)` ticker; `poll()` submits each msg as `worker.Job`; stops batch on `ErrPoolFull` (next tick retries)
- **Publisher interface**: `internal/outbox/outbox.go` → `Publisher`
- **Dev publisher**: `internal/outbox/log_publisher.go` → `LogPublisher` — logs and returns nil; replace with `KafkaPublisher` for production
- **Atomicity — Register**: `internal/customer/service.go` → `Service.Register()` — `repo.SaveTx(ctx, tx, c)` and `outboxStore.SaveTx(ctx, tx, msg)` share the same `pgx.Tx`, then `tx.Commit()`. Fully atomic.
- **Atomicity — Update/Remove (caveat)**: `repo.Update`/`repo.Delete` are called without a transaction; the outbox write uses a separate `tx`. These two operations are NOT atomic in the current implementation.
- **Post-commit**: after `tx.Commit`, `Service.appendEvent` writes to SQLite event log — best-effort, errors logged but not returned

### Worker Pool

- **Impl**: `internal/worker/pool.go` → `Pool`, `Job func(ctx context.Context) error`, `ErrPoolFull`
- **Lifecycle**: `New(ctx, workers, queueSize, logger)` starts goroutines immediately; workers exit on `ctx.Done()` or channel close
- **Backpressure**: `Submit()` is non-blocking; returns `ErrPoolFull` when buffered channel is full
- **Shutdown**: `Stop()` closes the job channel then `wg.Wait()` — no work is dropped; always call `Stop()` after context cancellation

### CQRS (explicit, no mediator)

- **Write**: `internal/customer/service.go` → `*Service` — side effects: writes DB, outbox, event log, invalidates cache
- **Read**: `internal/customer/query.go` → `*QueryService` — no side effects; no event log; no cache writes
- **Cached read**: `internal/customer/query.go` → `*CachedQueryService` — wraps `*QueryService`; the underlying `QueryService` is unchanged
- **Handler accepts interface**: `internal/customer/handler.go` → `querier` interface satisfied by both; wired to `*CachedQueryService` in `main.go`

### Domain Error → HTTP Mapping

- **Sentinel errors — customer**: `internal/customer/domain.go` → `ErrNotFound`→404, `ErrEmailExists`→409, `ErrInvalidBirthDate`→422
- **Sentinel errors — auth**: `internal/auth/domain.go` → `ErrEmailExists`→409, `ErrInvalidPassword`→401, `ErrPasswordTooShort`→422
- **Mapping**: `mapDomainError()` in `handler.go` of each package; `errors.Is()` switch → `echo.NewHTTPError(statusCode, err.Error())`

### sqlc + Repository Adapter

- **SQL source**: `queries/customers.sql`, `queries/users.sql`
- **Config**: `sqlc.yaml` — engine postgresql, pgx/v5, `emit_interface: false`; uuid override to `github.com/google/uuid.UUID`
- **Generated**: `internal/platform/database/postgres/*.sql.go`, `models.go`, `db.go` — never edit by hand
- **Adapter**: `internal/platform/database/postgres/customer_repo.go` — translates `pgtype.Date`/`pgtype.Timestamptz` ↔ `time.Time`; translates `pgx.ErrNoRows` → domain sentinels; `WithTx(tx)` for transactional writes
- **Interface satisfied implicitly**: `*CustomerRepository` satisfies `customer.Repository`; `*UserRepository` satisfies `auth.Repository`

## 10. Wiring Sequence (`main.go` order)

```
1.  config.Load()                           — viper + godotenv; os.Exit(1) if DATABASE_URL or JWT_SECRET missing
2.  newLogger(cfg.Env, cfg.LogLevel)        — JSON in prod ("production"/"prod"), text otherwise; slog.SetDefault
3.  signal.NotifyContext(SIGINT, SIGTERM)   — root ctx; cancellation drives entire shutdown
4.  database.NewPool(ctx, cfg.DatabaseURL)  — pgxpool, pings; os.Exit(1) on failure
5.  runMigrations(cfg.DatabaseURL)          — goose.Up over pgx driver; os.Exit(1) on failure
6.  os.MkdirAll("data", 0750)              — creates directory for SQLite file
7.  eventlog.NewSQLiteStore(cfg.EventLogPath) — opens SQLite, runs inline DDL migration
8.  cache.NoopCache{} or cache.NewRedisCache — conditional on cfg.RedisAddr != "";
                                              WARN + fallback to NoopCache on ping failure
9.  worker.New(ctx, cfg.WorkerCount, cfg.WorkerQueue, logger) — goroutines start immediately
10. outbox.NewPostgresStore(pool)           — no I/O at construction
11. outbox.NewLogPublisher(logger)          — no I/O at construction
12. outbox.NewPoller(store, publisher, pool, interval, batch, logger)
13. go poller.Run(ctx)                      — polling goroutine starts; exits on ctx.Done
14. postgres.NewCustomerRepository(pool)   — wraps sqlc Queries
15. customer.NewService(repo, pool, outboxStore, eventLog, cache, logger)
16. customer.NewCachedQueryService(customer.NewQueryService(repo), cache, cfg.CacheTTL, logger)
17. customer.NewHandler(svc, cachedQuery)
18. postgres.NewUserRepository(pool)
19. auth.NewService(userRepo, cfg.JWTSecret, cfg.JWTExpiry, logger)
20. auth.NewHandler(authSvc)
21. [if cfg.GRPCEnabled] customer.NewGRPCHandler(customerSvc, customerQuery, eventLog)
22. [if cfg.GRPCEnabled] grpcserver.NewServer(handler, authSvc, logger)
23. [if cfg.GRPCEnabled] go grpcSrv.Serve(lis)  — gRPC server goroutine
24. server.Start(ctx, cfg, customerHandler, authHandler, cache, workerPool, logger) — BLOCKS
25. [ctx cancelled] server.Start returns after Echo drains (10s timeout)
26. [if cfg.GRPCEnabled] grpcSrv.GracefulStop() — waits for in-flight gRPC calls
27. workerPool.Stop()                       — drains queued jobs; waits for in-flight
28. [defer] pool.Close()                    — Postgres connections released; process exits 0
```

## 11. Graceful Shutdown Sequence

```
Step 1: OS SIGINT/SIGTERM received
        → signal.NotifyContext cancels root ctx
        → poller.Run(ctx) exits select loop immediately — no more polls
        Reason: poller exits first so no new jobs are submitted after shutdown begins

Step 2: server.Start detects <-ctx.Done()
        → calls e.Shutdown(shutdownCtx) with 10-second timeout
        → Echo stops accepting new connections; drains in-flight HTTP handlers
        → server.Start returns
        Reason: drain HTTP first so handlers can still call service/DB while pool is live

Step 3: workerPool.Stop() called in main()
        → close(pool.jobs) signals workers to finish after current job
        → p.wg.Wait() blocks until all goroutines finish
        → in-flight outbox delivery jobs complete (with cancelled ctx passed to Publish)
        Reason: outbox jobs must finish before DB connections are released

Step 4: defer pool.Close() executes
        → pgxpool releases all Postgres connections
        Reason: last because steps 2–3 may still issue DB queries

Step 5: process exits 0
        Note: SQLite db.Close() is not called explicitly in main — the OS closes the
        file descriptor on exit. Add explicit Close if crash-safety is required.
```

## 12. Testing Conventions

### Unit Tests
- Build tag: none — included in `go test ./...`
- Package: `package customer_test` (external test package pattern)
- Mock strategy: hand-crafted stubs in test files for existing tests; `mockery`-generated for new work
- Examples: `internal/customer/customer_test.go` (table-driven TestNew + TestService_Register), `internal/worker/pool_test.go`, `internal/outbox/poller_test.go`
- Canonical table-driven example: `TestNew` in `customer_test.go` — `[]struct{name, inputs, wantErr}`, `t.Run(tc.name, ...)`, `t.Parallel()` inside subtests

### Integration Tests
- Build tag: `//go:build integration` — excluded from `go test ./...`
- Run: `make test-integration` → `go test ./... -tags=integration -race -count=1`

## 15. What NOT to Do

- **Do NOT define interfaces in `platform/database/postgres/`** — interfaces belong at the consumer; the infra package satisfies them implicitly
- **Do NOT import `platform/database/postgres` from any domain package** — dependency arrows point inward; infra imports domain, never the reverse
- **Do NOT use GORM or any ORM** — `sqlc` generates type-safe Go from SQL; edit `queries/*.sql` and run `make generate`
- **Do NOT use `uber-go/fx`, `google/wire`, or any DI framework** — all wiring is in `main.go`; if it exceeds ~80 lines, split into `cmd/api/wire.go`
- **Do NOT create a mediator, command bus, or event bus** — call service methods directly; async dispatch uses `worker.Pool.Submit()`
- **Do NOT put business validation in handlers** — `go-playground/validator` handles request shape; domain rules live in `customer.New()`, `customer.Update()`, and service methods
- **Do NOT log the same error at multiple layers** — wrap with `fmt.Errorf("pkg.Type.Method: %w", err)`, log once at the handler boundary
- **Do NOT use `interface{}` or `any` in domain structs** — use concrete types; generics if truly needed
- **Do NOT add `panic()` to domain, service, or handler code** — return errors always
- **Do NOT skip `-race` in tests** — `make test` always passes `-race -count=1`
- **Do NOT edit `*.sql.go`, `models.go`, or `db.go`** — re-run `make generate` after changing SQL
- **Do NOT call `outboxStore.SaveTx` outside a transaction** — it takes `pgx.Tx`; writing outside a tx defeats atomicity
- **Do NOT hand-write mocks for new interfaces** — generate with `mockery`
- **Do NOT share domain types or data stores across bounded contexts** — `customer` and `auth` are isolated; `product` (future) must be independent
- **Do NOT write Portuguese in code, comments, commits, or docs**
